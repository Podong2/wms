side nav

  • 사용자는 숨어있는 side menu를 특정버튼을 눌러 노출시킬 수 있다.
  • 노출된 side menu는 content영역을 밀어서 노출이 아닌 content위로 노출 되어야한다.
  • 노출된 메뉴는 메뉴 이외의 영역을 누르거나 메뉴명을 누를 시 화면에서 사라진다.
  • 메뉴는 트리형태의 구로조 표현 가능해야한다.
  • 선택된 메뉴는 active 효과를 적용해야한다.
  • 사이드 바를 고정도 가능해야한다.
  • jhipster처럼 메뉴 오픈 시 content영역이 밀리거나 덮히는 옵션이 있어야한다.
  • intellij의 우측메뉴와 동일하게 각 메뉴의 정보가 노출되어야한다.

select box

    - 옵션 설명 -
    ng-model : select box에서 보여질 데이터 list
    selectedValue :  select box에서 선택된값이 담길 scope명
    actionsBox : 전체선택, 전체 해제를 선택할 수 있는 버튼 노출 유무 예) true - 노출
    multiple : 해당 옵션을 주입하면 multi select 기능을 사용할 수 있다.
    dataSize : option 창에서 화면에 노출될 data 개수
    dataLiveSearch : filter 기능 사용 유무 선택 예) true - 노출
    disabled : 해당 옵션 주입 시 특정 scope 값을 이용하여 사용/미사용을 선택할 수 있다. 예) true - 사용

    * 아이콘 사용 시 array 형식 *
        - 아이콘 사용시에는 "icon" key에 노출시킬 아이콘 class값을 주입한다.
        vm.arrayCollection = [
            {name: 'Alice', class: 'Class A', icon: 'glyphicon-euro'},
            {name: 'Bob', class: 'Class B', icon: 'glyphicon-usd'}
        ];

Toast

  • 시스템의 메시지 노출시에는 toast로 노출한다.
  • 노출 후 특정 시간이 지난 후 자동으로 닫힌다.
  • 오류 메시지는 자동닫힘이 없고 x 버튼을 눌러야 닫힌다.
  • 메시지별 각각의 컬러 테마가 적용된다.

  • 표현되는 text는 개행되지 않아야한다.

datePicker

  • 좌우 화살표 또는 월을 선택하여 원하는 달의 달력을 확인할 수 있다.
  • 달력은 자동으로 오늘날짜를 표시해야한다.
  • 월별로 날짜를 출력하고 해당 날짜를 클릭하여 달력값을 얻을 수 있어야한다.
  • 다국어 가능해야한다.
  • input에서 바로 입력이 가능해야한다.
  • 구간검색이 가능해야한다
  • 단순 입력이 가능해야한다.
  • 년단위로 달력이동가능
  • 특정기간만 선택 가능하도록 가능
  • 구간검색이 가능해야한다
  • 시작일과 종료일에 대한 validation check를 하여 메세지 노출 혹은 자동 기입
  • 시작일자는 종료일자보다 작아야한다.
  • 시작일자는 종료일자와 같을수있다.
  • 특정기간만 선택가능하다.

Modal

  • 기존 owl과 동일한 modal이 적용되야한다.
  • 드래그로 창을 이동 가능해야한다.
  • 엔터키 입력 시 창의 전송기능 작동 가능해야한다.
  • esc 입력시 창이 닫힌다.
  • 배경색은 투명색으로 유지한다.
  • 내용이 많을 시 화면이 벗어나지 않는 선에서 스크롤 생성되야 한다.

ngGallery

  • 이미지 확장자명의 파일은 갤러리로 표현이 가능해야한다.
  • 이미지 리스트는 해당 이미지의 썸네일로 표시되고 이미지를 클릭 시 겔러리 화면에서 해당 이미지의 원본 이미지를 볼 수 있다.
  • 갤러리에서는 다음 이미지 혹은 이전 이미지로 이동할수있는 링크가 존재한다.
  • 바로 이동가능한 썸네일 이미지 목록이 존재해야한다.
